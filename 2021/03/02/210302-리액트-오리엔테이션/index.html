commit 50/72 규칙

  • 제목은 50자 내외 ( 마침표(.) 사용 ✘ )
  • 제목과 본문은 빈 줄로 구분
  • 본문은 한 줄에 72자 안으로 작성
  • 본문은 “무엇”을, “왜”, “어떻게” 작성했는 지 기술
  • setting.json 에서,
  • 커밋 플래그
    추가 (Added) - 새로운 내용이 추가된 경우
    변경 (Changed) - 기존 기능이 변경된 경우
    중단 예정 (Deprecated) - 더 이상 지원 예정이 없어 제거될 경우
    수정 (Fixed) - 버그가 발견되어 수정된 경우
    취약점 (Security) - 취약점이 발견된 경우

alias

  • https://git-scm.com/book/ko/v2/Git%EC%9D%98-%EA%B8%B0%EC%B4%88-Git-Alias
  • Git의 명령을 전부 입력하는 것이 귀찮다면 git config 를 사용하여 각 명령의 Alias을 쉽게 만들 수 있다.
  • git 글로벌 설정 파일에서 수정할 수도 있다.

스니펫

  • 재사용 가능한 코드 조각을 간단한 단축키로 저장.
  • https://snippet-generator.app/ 에서 작성 후,
  • 설정 -> 사용자 코드 조각 -> 원하는 파일 검색후 복붙.
