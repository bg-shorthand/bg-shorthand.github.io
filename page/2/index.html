201019-What is HTML?

Posted on 2020-10-19 In html/css

HTML5 MARKUP

  1. MAKRUP
    • 마크한다는 의미.
    • 문서에 태그를 추가하여 정보를 입력
  2. HTML5
    • MARKUP + API
    • 마크업 언어면서, API가 추가된 최신의 HTML.
  3. HTML4.0, XHTML1.0, HTML5
    • 세 가지 모두 표준으로 채택된 상태.
    • HTML은 기본적으로 룰셋이 느슨하다.
    • 룰셋이 훨씬 엄격한 xml. 그렇게 만들어진 XHTML.
  4. HTML5 이해하기
    1. 콘텐츠 모델
      • 시멘틱 태그들을 일정 콘텐츠(-카테고리)로 연관지을 수 있음.
      • 가령, 섹션이나 인터렉티브.
      • 태그들을 구분하여 사용.
    2. 아웃라인 알고리즘
      • HTML5 이전에는 암묵적인 아웃라인이 있었다.
      • HTML5부터 명시적인 아웃라인 선언.
      • HTML 문서가 목차 형태의 구조를 갖게 됨.
  5. HTML5는 구조를 만드는 것이다.
    • 웹페이지에 보이는 구조와 별개로, 구조적인 관점에서 생각해야 함.
    • 논리적인 순서를 고민해서 기획/설계.
    • 시멘틱 마크업을 통해 의미를 분명하게 넣는다.
    • ex) h1>a>img{logo} -> 이 로고는 이 문서의 h1(대제목)에 해당함을 의미.
    • 클래스 등의 네이밍 또한 의미를 생각하여 만들어야 한다.
    • 단순히 웹사이트를 렌더링하기 전에, 설계가 선행되어야 한다!

201016 How to git

Posted on 2020-10-18 Edited on 2020-10-19 In git/github

git bash

  1. $ 표시가 있으면 git bash가 입력 받을 준비가 된 상태.
  2. ~ 는 시스템 최상단의 폴더
  3. 맨 앞의 hanha@DESKTOP 는 ‘유저이름‘@’컴퓨터이름’
  4. 명령어
    • ls : 현재 디렉토리 안의 파일과 디렉토리 표시.
    • mkdir : (make directory) 디렉토리 만들기
    • touch : 파일 만들기
    • pwd : 현재 디렉토리의 절대 경로 표시
    • mv : (move) 이동
    • cp : (copy) 복사. mv처럼 사용. 확장자도 변경 가능.
    • rm : (remove) 파일 삭제
    • ctrl+c : 명령 중단
    • code (파일명) : vs code 로 해당 파일 실행.
    • vi (파일명) : vim으로 해당 파일 실행.

vim

  1. 명령어
    • i : 입력모드 전환
    • esc : 노멀모드 전환
    • : : 입력모드에서, 메뉴바 역할

gui shall : 탐색기(파인더)
object : 파일
path : 디렉토리
cl -l로 파일 상세 정보 읽었을 때, 맨 앞의 문자열의 의미

  • ex) drwxrwxrwx
  • 맨 앞의 d가 있으면 디렉토리, -면 파일
  • 그 뒤로 세글자씩 잘라서 rwx/rwx/rwx = 나/나를포함한그룹/게스트 의 읽고 쓰고 실행할 수 있는 권한.
  • rwx 각각은 1이나 0을 표현(2진수). ex) rwx = 111(2), r-x = 101(2)
  • rwx 묶음은 다시 8진수로 표현 가능. ex) rwx = 111(2) = 7(8), r-x = 101(2) = 5(8)
  • 즉, rwxrwxrwx 는 777로 표현 가능.
  • 이걸 조작할 수 있는 명령어가 chmod.(윈도에서 동작 안함.)

git (=Version Control System)

  1. 과정
    • git init 혹은 git clone 으로 로컬 리포지토리 생성
    • 로컬에서 작업
    • git add 로 추적되지 않은 파일을 스테이지에 업로드
    • git commit 으로 업로드된 파일 목록을 커밋
    • git push 로 github의 리포지토리로 업로드
  2. 명령어
    • git status : 깃 폴더의 상태 표시
    • git status -uall
    • git remote -v
    • git remote get url origin
